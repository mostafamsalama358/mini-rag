# .env.example

## What is this?

Example **environment variables** for local development (without Docker).

Copy to `.env` and add your real API keys.

## Why does it exist?

The app never hard-codes secrets. All config comes from env vars.

`src/helpers/config.py` reads these names.

## Main sections

| Section | Examples |
|---------|----------|
| App | `APP_NAME`, `APP_VERSION` |
| Files | `FILE_ALLOWED_TYPES`, `FILE_MAX_SIZE` |
| PostgreSQL | `POSTGRES_HOST`, `POSTGRES_PASSWORD` |
| LLM | `OPENAI_API_KEY`, `COHERE_API_KEY`, model IDs |
| Vector DB | `VECTOR_DB_BACKEND` = `PGVECTOR` or `QDRANT` |
| Language | `PRIMARY_LANG`, `DEFAULT_LANG` |
| Celery | `CELERY_BROKER_URL`, `CELERY_RESULT_BACKEND` |

## Local vs Docker

| Setting | Local (this file) | Docker (`docker/env/.env.example.app`) |
|---------|-------------------|----------------------------------------|
| Postgres host | `localhost` | `pgvector` |
| Ports | Standard local ports | Docker network names |

## Where is it used?

| File | How |
|------|-----|
| `src/helpers/config.py` | `Settings` class loads `.env` |
| `src/flowerconfig.py` | Reads `CELERY_FLOWER_PASSWORD` |
| Root `README.md` | `cp .env.example .env` |
| `docker/env/.env.example.app` | Same variables for containers |
