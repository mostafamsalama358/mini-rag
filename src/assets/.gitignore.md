# .gitignore

## What is this?

Tells Git to **ignore uploaded user data** in the assets folder.

## Why does it exist?

When users upload files, they go to:

- `assets/files/` — uploaded PDFs and TXT files
- `assets/database/` — local database files

This data should stay on your machine, not in Git.

## What is ignored

```
files
database
```

Both folders are created at runtime by controllers.

## Where is it used?

| File | How |
|------|-----|
| `src/controllers/BaseController.py` | Uses `assets/files` and `assets/database` paths |
| `src/controllers/ProjectController.py` | Creates `assets/files/{project_id}/` |
| `docker/docker-compose.yml` | `fastapi_data` volume mounts `/app/assets` |

## .NET comparison

Like ignoring `wwwroot/uploads/` in a web app.
